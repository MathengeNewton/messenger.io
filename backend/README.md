## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Running church360 APIs with Docker

church360 ships with full Docker and Docker Compose support for easy local development and deployment.

### Prerequisites
- [Docker](https://docs.docker.com/get-docker/) and [Docker Compose](https://docs.docker.com/compose/install/) installed on your machine.

### 1. Clone the repository
```bash
git clone https://github.com/MQ9Realper/church360Backend.git
cd church360Backend
```

### 2. Configure Environment Variables
Copy the docker environment file and edit as needed:
```bash
cp .env.docker .env
```
- Set your database, JWT, and other secrets in `.env`.

### 3. Build and Start the Containers
```bash
docker-compose up --build
```
- This will build the backend image and start both the NestJS API and PostgreSQL database containers.

### 4. Accessing the APIs
- The backend API will be available at: [http://localhost:3000](http://localhost:3000)
- Swagger API docs: [http://localhost:3000/api/docs#/](http://localhost:3000/api/docs#/)

### 5. Stopping the Containers
```bash
docker-compose down
```

### 6. Common Docker Commands
- Rebuild after code changes: `docker-compose up --build`
- View logs: `docker-compose logs -f`
- Run migrations: `docker-compose exec backend npm run typeorm:migration:run`

### Troubleshooting
- If you get `crypto is not defined` or build errors, ensure you are using Node 18+ and the latest Docker images.
- For persistent database storage, check the `docker-compose.yml` volumes section.
